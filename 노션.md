# 노션 배포

`data/listings.json`의 데이터를 Notion 페이지에 표로 그리는 방법(push)과 인증.
동기화는 **양방향**이다: `make_notion.py`가 JSON을 Notion으로 내보내고(push), `notion_pull.py`가 Notion 편집을 JSON으로 회수한다(pull). Notion에서 직접 편집할 수 있으므로, **배포(push) 전에는 항상 pull을 먼저 실행**해 직접 편집분을 회수한다.
토큰 등 인증 정보는 `.notion.json`을 참조한다.

## 라이브 산출물

- **Notion "주거 계획" 페이지**: https://www.notion.so/371331f433e280ff9848ea756d30aae6
  - 페이지 ID: `371331f433e280ff9848ea756d30aae6`
  - 2단계 구성: **거래유형**(`매매` / `월세/전세`)을 `heading_2`로, 그 아래 **'시'**(예: `경기도 용인시`)를 `heading_3`로 두고, 시마다 단순 표(simple table) 1개. 매물 있는 시만 표가 생긴다.
  - 표 구성: 전치 매트릭스(매물=열, 속성=행), 1열=속성 라벨(좌상단은 빈칸), 매물 이름은 굵게, 장점/단점은 셀 안에 글머리로 줄바꿈.
  - 컬럼 구성은 거래유형마다 다르다(`config.json`의 `groups[].attrs`): 매매는 `가격(억)`·`토허제 해당`을 쓰고, 월세/전세는 `가격` 대신 `월세 가격`·`전세 가격`을 쓰며 `토허제 해당`이 없다. 같은 거래유형의 시 표들은 컬럼이 같다.
  - Notion '데이터베이스'는 항목=행이 고정이라 가로 배치가 불가능해, 정적 **표 블록**으로 매물=열의 가로 레이아웃을 재현한다.

## 표 갱신 (pull → 편집 → push)

```bash
cd scripts
python3 notion_pull.py       # ① (필수) Notion 직접 편집분을 data/*.json으로 먼저 회수
python3 make_notion.py       # ② 부모 페이지에 거래유형>시별 표를 (재)생성
```
- **반드시 pull을 먼저** 돌린다. 건너뛰고 push하면 Notion에서 직접 고친 내용이 덮어쓰여 사라진다.
- push는 이전에 만든 표/헤딩 블록(`.notion.json`의 `block_ids`로 추적)을 **삭제하고 새로 그린다.** 중복이 생기지 않고 페이지 URL은 유지된다.
- `pull → 편집 → push` 사이에 발생한 Notion 직접 편집은 회수되지 않으므로, 작업 창을 짧게 유지한다.

## pull (Notion → JSON)

```bash
cd scripts
python3 notion_pull.py
```
- Notion 표를 역파싱해 `data/listings.json`·`data/config.json`에 **merge**한다.
- Notion 표에 있는 필드(가격·장단점 등)만 갱신하고, **Notion에 없는 로컬 필드(`id`, `estimate`, 향후 지도 좌표)는 보존**한다.
- 표에서 빠진 매물은 자동 삭제하지 않고 경고만 한다. 표에 새로 생긴 매물은 `id` 없이 추가되며, `id`를 수동 부여하라고 경고한다.
- 매물명을 키로 매칭하므로, **Notion에서 매물명 자체를 바꾸면** 기존 `id`·좌표 연결이 끊긴다(신규로 인식). 이름 변경은 JSON에서 하는 것을 권장.

## 인증

`make_notion.py`는 `../.notion.json`(또는 환경변수)에서 자격 증명을 읽는다.
```json
{
  "token": "ntn_...",            // Notion integration 토큰 (필수)
  "parent_page_id": "...",       // 표를 넣을 페이지. URL 통째로 넣어도 됨.
  "block_ids": [...]             // 자동 기록 — 이전에 만든 표/헤딩 블록 id (재실행 시 교체용)
}
```
환경변수 `NOTION_TOKEN` / `NOTION_PARENT_PAGE_ID`로도 줄 수 있다.

### 통합 토큰 발급 (최초 1회, 사람만 가능)
1. https://www.notion.so/my-integrations → **New integration** (유형 **Internal**) → 저장
2. **Internal Integration Secret**(`ntn_...`) 복사 → `.notion.json`의 `token`에 저장
3. **⭐ 페이지 공유(필수)**: 부모 페이지 → 우상단 **⋯ → 연결(Connections)** → 만든 통합 추가.
   (이걸 빠뜨리면 토큰이 유효해도 `object not found` 오류가 난다.)

### 보안 주의 (중요)
- `.notion.json`은 **절대 커밋 금지**(`.gitignore` 등록됨).
- 토큰이 노출되면 https://www.notion.so/my-integrations 에서 **Refresh secret(재발급)** 후 `.notion.json`만 갱신한다.
